## Grok  _(grok-build, 49.3s)_
**The error is here:**

> "Equivalently, all pairwise *differences* must be distinct (a **Golomb ruler**)."

This is false. Distinct pairwise sums (a_i + a_j for i < j) is a strictly weaker condition than all pairwise |differences| being distinct.

- When two pairs share an element (e.g., consecutive differences of 1 forming 1,2,3), the "cross sum" that would collide is of the form b + b or similar and is never generated by distinct-pair sums.
- Therefore the pigeonhole argument "d >= 11 because 10 distinct positive differences must fit in 1..d" does not apply. Only four-distinct-element additive collisions are forbidden.

The proposed set {1,2,5,10,12} satisfies the (stronger) Golomb condition and therefore works, but it is far from optimal for the actual problem.

**Better bound and result**

The sums lie between 1+2=3 and (M-1)+M=2M-1. At least 10 distinct values are needed in a range of size 2M-3, so M >= 7 is the information-theoretic floor. Brute-force enumeration over all 5-subsets shows:

- No solution exists for any M <= 7.
- Solutions exist for M = 8; one is {1,2,3,5,8}.

**Verification for [1,2,3,5,8]**

Pairwise sums: 3,4,5,6,7,8,9,10,11,13 — exactly 10 distinct values. Largest element = 8.

This is optimal (minimal possible largest element).
